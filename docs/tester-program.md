# WizBlock Google Play Tester Program

WizBlock is recruiting testers for the Google Play closed-testing track.
Testing helps validate installation, onboarding, Accessibility setup, overlay
blocking, schedules, usage limits, and Strict Mode on different Android
devices.

## How To Join

1. [Email kirentps@gmail.com](mailto:kirentps@gmail.com?subject=WizBlock%20Google%20Play%20tester)
   with the Google account email address you want to use for testing.
2. We will add that address to the private Google Play tester list.
3. We will reply with the Google Play opt-in link when the tester list is ready.
4. Open the opt-in link while signed in to the same Google account, then install
   WizBlock from Google Play.
5. Use the app during the testing period and report useful feedback through
   [GitHub Issues](https://github.com/kirentanna/WizBlock/issues).

> Please do not post your Google account email publicly in a GitHub issue.

## What To Test

- Fresh installation and onboarding.
- Accessibility and overlay permission disclosures.
- Blocking an app, website, or keyword.
- Blocklist schedules and selected days of the week.
- Daily usage limits and timed focus sessions.
- Strict Mode and its optional uninstall-protection flow.
- Recovery after restarting the phone or reopening WizBlock.

When reporting a problem, include the Android version, device model, WizBlock
version, and steps to reproduce. Do not include personal data or screenshots
containing private notifications.

## Testing Notes

WizBlock is local-first: rules, schedules, usage counters, and block history
remain on the device. It does not require an account, does not include ads or
analytics SDKs, and does not request internet access.

The closed-testing track is separate from the downloadable APK on GitHub. The
Google Play tester link installs the Play-distributed build; the GitHub Release
is an optional direct-download channel.
